# Titolo della Pull Request

## Descrizione
Breve descrizione del problema o della funzionalità che questa PR risolve o introduce.

## Motivazione
Spiega perché questa PR è necessaria. Qual è il contesto o la ragione dietro questa modifica?

## Cosa è stato fatto
Descrivi le modifiche apportate nel dettaglio. Puoi includere snippet di codice, immagini o diagrammi se necessario.

## Come testare
Fornisci istruzioni su come testare le modifiche apportate da questa PR. Ad esempio, quali comandi eseguire o quali scenari verificare.

## Verifica
- [ ] Ho testato le modifiche localmente.
- [ ] Ho verificato che i test passino.
- [ ] Ho aggiornato la documentazione se necessario.

## Note aggiuntive
Eventuali note o considerazioni extra.
